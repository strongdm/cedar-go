# Sprint Documentation

This directory contains sprint planning documents for cedar-go development.

## Structure

- `SPRINT-NNN.md` - Final approved sprint documents
- `drafts/` - Working drafts and planning artifacts
  - `SPRINT-NNN-INTENT.md` - Concentrated intent for planning
  - `SPRINT-NNN-CLAUDE-DRAFT.md` - Claude's draft
  - `SPRINT-NNN-CODEX-DRAFT.md` - Codex's competing draft
  - `SPRINT-NNN-CLAUDE-DRAFT-CODEX-CRITIQUE.md` - Codex's critique
  - `SPRINT-NNN-MERGE-NOTES.md` - Synthesis notes

## Sprint Format

Each sprint document includes:
- **Overview**: Why and high-level approach
- **Use Cases**: What users can do
- **Architecture**: System design
- **Implementation Plan**: Phased tasks
- **Files Summary**: What's created/modified
- **Definition of Done**: Success criteria
- **Risks & Mitigations**: What could go wrong
- **Dependencies**: What's needed
